# ✈️ 어케가지 (akegazi)

> **"어케 가지?"** — 날짜별 목적지만 넣으면 **구글 지도 검색·동선·시간 일정·날씨·길찾기**를 한 번에 정리해 주는 여행 플래너.

서버·빌드가 없는 **정적 웹앱**이라 GitHub Pages에 그대로 올라갑니다. 구글 지도 검색·동선은 **브라우저용 구글맵 키 1개**(`js/config.js`, 리퍼러 제한)로 동작합니다.

**▶ 데모: https://nj1099.github.io/akegazi/**

## 핵심 기능

| 기능 | 설명 |
|------|------|
| 🧳 여행별 관리 | **나라·일정별로 여러 여행**을 만들어 관리. 홈(여행 목록) → 여행(날짜) → 날짜(장소) 3단계. |
| 📅 일자별 일정 | 날짜를 추가하고, 그날 장소를 **가고 싶은 곳 · 먹고 싶은 곳 · 카페 · 체험 · 숙소 · 이동 · 공항**으로 분류해 타임라인으로 봅니다. |
| ↕️ 드래그 정렬 | 타임라인에서 **손잡이(⠿)를 끌어** 장소 순서를 바꿉니다(터치·마우스). 장소/여행 삭제, 전체 지우기도. |
| 🔎 구글 지도 검색 | 장소 이름·주소를 **구글 Places**로 검색(입력 즉시 후보 표시). 키 없으면 OSM 폴백. |
| 🧭 동선 최적화 | 좌표가 있는 장소를 **nearest-neighbor + 2-opt**로 재정렬. 공항·숙소·고정 일정은 자리 보존. |
| ✈️ 공항 시간 일정 | 공항에 **도착·출발 시각**을 넣으면 그날 목적지의 **예상 도착(ETA)을 자동 계산**. 도착편은 하루 시작, 출발편은 마감으로 보고 **비행기 놓칠 위험을 빨간 배너**로 경고. |
| 🌦 실시간 날씨 | **Open-Meteo**(무료·키 불필요)로 일자별 기온·강수·확률. 미래 16일 예보 + 과거 실측. |
| ☔ 우천 → 실내 추천 | 강수 임계 초과 시 **‘실내 위주’ 배너** + 야외 일정에 ‘우천주의’ 배지. |
| 🍴 휴무·예약 체크 | 휴무 요일이 방문일과 겹치면 **자동 경고**, 예약 불필요/권장/필수/완료 배지. |
| 🗺 구글맵 동선 | 일자별 동선을 **방문 순서 번호 마커 + 점선**으로 다크 스타일 **구글맵**에 표시. |
| 🚇 길찾기 | ‘길찾기(이전→여기)’·‘전체 길찾기’가 **구글맵 대중교통 경로**로 연결. |
| 📤 일정 공유 | 일정을 **링크 하나로** 카카오톡·텔레그램 등에 공유. 받은 사람은 링크만 열면 그대로 불러옵니다. |
| 💾 저장·백업 | 브라우저 **localStorage** 자동 저장 + JSON 가져오기/내보내기. |

## 사용법

1. 홈에서 **`+ 새 여행`** 으로 여행을 만들거나(나라·일정별), **`예시`** 로 오사카 3일을 불러옵니다.
2. 여행을 열어 **`+ 날짜 추가`** → 날짜를 열고 **`+ 장소 추가`** → 이름·종류·위치(구글 검색/지도클릭)·시간·체류시간·영업시간·휴무·예약·실내여부·메모 입력. **공항**은 도착·출발 시각을 함께 넣으세요.
3. 타임라인에서 **손잡이(⠿)를 끌어 순서**를 바꾸고(드래그), **`✕`** 로 장소를 지웁니다. **`동선 최적화`** · **`지도`** 탭(번호 마커)으로 동선 확인. 공항 시각을 넣으면 **구간별 예상 도착 시각**과 **비행기 마감 경고**가 자동으로 떠요.
4. 각 카드 **`길찾기`** 로 구글맵 경로를, 상단 **`↗ 공유`** 로 여행을 친구에게 보냅니다.
5. 앱 제목 **`어케가지`** 를 누르면 홈(여행 목록)으로, **`←`** 로 한 단계 뒤로 갑니다.

## 다른 사람과 함께 쓰기

- **그냥 같이 쓰기**: 누구든 **https://nj1099.github.io/akegazi/** 주소만 열면 바로 사용합니다(설치 불필요). 각자의 데이터는 자기 브라우저에만 저장되어 서로 섞이지 않습니다.
- **내 일정을 보내주기**: 상단 **`↗ 공유`** → 휴대폰이면 카카오톡·텔레그램 공유 시트가 뜨고, PC면 링크가 복사됩니다. 받은 사람이 링크를 열면 “공유받은 일정 — 불러올까요?” 안내가 떠 그대로 자기 플래너로 가져갑니다.
- **참고(실시간 공동 편집)**: 여러 명이 같은 일정을 **동시에 실시간으로** 고치는 건 서버가 필요해 현재 정적 배포에서는 지원하지 않습니다. 대신 “공유 링크로 주고받기”나 JSON 내보내기/가져오기로 일정을 공유·동기화하세요. (원하면 추후 소규모 백엔드를 붙여 공동 편집을 추가할 수 있습니다.)

## 기술 스택 (빌드 없음)

- **순수 바닐라 JS** — 클래식 스크립트 + 전역 네임스페이스 `TP`.
- **지도·검색**: [Google Maps Platform](https://developers.google.com/maps) — Maps JavaScript API(다크 스타일 + 번호 마커) + Places API(New) Text Search. 브라우저 키 1개(리퍼러 제한). 동적 라이브러리 임포트로 필요 시 지연 로드.
- **시간 일정**: `geo.buildSchedule` — 공항 도착/출발 시각 기준 구간 ETA(이동시간은 직선거리/22km·h 추정) + 비행기 마감 경고.
- **날씨**: [Open-Meteo](https://open-meteo.com/) Forecast/Archive (무료·키 불필요)
- **지오코딩 폴백**: [Nominatim](https://nominatim.org/)(OSM) → Open-Meteo Geocoding (키 없을 때만)
- **길찾기**: Google Maps Directions URL 딥링크
- **공유**: 일정을 URL-safe base64로 해시(`#trip=`)에 담아 **Web Share API**(모바일) / 클립보드(PC)

## 구글맵 키 설정

1. [Google Cloud Console](https://console.cloud.google.com)에서 프로젝트 생성 → **결제 계정 연결**(개인 사용은 매월 무료 한도 내).
2. **Maps JavaScript API**·**Places API (New)** 사용 설정 → **API 키** 발급.
3. 키 제한(필수): **애플리케이션 제한 = HTTP 리퍼러**(`https://nj1099.github.io/*`, `http://localhost:*/*`), **API 제한 = 위 두 API**.
   - ⚠️ **리퍼러는 반드시 origin 와일드카드 `https://nj1099.github.io/*`** 로 등록하세요. 검색(Places API New)은 브라우저 정책상 **origin(`https://nj1099.github.io/`)만** 리퍼러로 보내므로, `/akegazi/*` 처럼 경로에 한정하면 **지도는 되지만 검색이 403**으로 막힙니다.
4. 발급한 키를 `js/config.js`의 `GOOGLE_MAPS_API_KEY`에 넣습니다. (리퍼러 제한 브라우저 키라 공개돼도 타 도메인에선 동작하지 않습니다.)
   비워두면 검색은 OSM 폴백, 지도는 안내 메시지로 동작합니다.

## 프로젝트 구조

```
akegazi/
├─ index.html          # SPA 셸 (config→gmaps→모듈 로드)
├─ css/app.css         # 다크 프리미엄 디자인 시스템
└─ js/
   ├─ config.js        # 구글맵 API 키(리퍼러 제한 공개 키)
   ├─ gmaps.js         # 구글맵 동적 로더 + TP.gmaps.lib() 헬퍼
   ├─ util.js          # DOM/날짜/fetch/토스트 헬퍼
   ├─ store.js         # Trip→Day→Stop 모델 + localStorage + import/export
   ├─ geo.js           # 지오코딩(구글 Places) · 거리 · 동선 최적화 · buildSchedule · 딥링크
   ├─ weather.js       # Open-Meteo + 우천 시 실내 추천
   ├─ maps.js          # 구글맵 JS API: 번호 마커·점선 동선·위치 선택기
   ├─ render.js        # 타임라인/카드/배지/날씨·일정 배너
   ├─ editor.js        # 장소/날짜 추가·편집 모달(공항 시각·체류·라이브 검색)
   ├─ share.js         # 일정 링크 공유 + 공유 링크 열기 시 불러오기
   ├─ sample.js        # 예시(오사카 3일) 데이터
   └─ app.js           # 라우팅 · 오케스트레이션 · 날씨/일정 로딩
```

## 로컬 실행

```bash
python -m http.server 4488   # → http://127.0.0.1:4488/
```
(로컬에서 구글 검색·지도를 테스트하려면 키 리퍼러에 `localhost`가 포함돼야 합니다.)

## 배포 (GitHub Pages)

이미 `main` 브랜치 루트로 배포되어 있습니다. 코드를 고친 뒤 **`git push`** 하면 자동 재배포됩니다.
(저장소 Settings → Pages: Source = *Deploy from a branch*, Branch = `main` / `(root)`, `.nojekyll` 포함)

## 개인정보

- 일정 데이터는 **브라우저에만** 저장됩니다(localStorage). 서버로 전송하지 않습니다.
- 공유 링크는 일정 내용을 **링크 자체에** 담아 전달합니다(서버 저장 없음). 외부 호출은 구글맵(지도·검색)·날씨뿐이며, 검색어/좌표만 해당 서비스로 전송됩니다.
